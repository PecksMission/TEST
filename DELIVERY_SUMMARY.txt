ARCHITECTURE OVERVIEW
================================================================================

USER FLOW:
  Chat Interface
       ↓
  Claude Multi-turn Conversation
       ↓
  Extract JSON Structure
       ↓
  Save to Database
       ↓
  Publish Timeline Page
       ↓
  Share / Embed / Comment

DATABASE:
  health_journeys (main table with AI-generated content as JSON)
  journey_comments (support comments and reactions)

API ENDPOINTS:
  GET  /journey/builder              - Chat interface
  POST /api/journey/chat             - Process message
  POST /api/journey                  - Create journey
  GET  /health-journey/<slug>        - View published
  GET  /api/journey/<id>/comments    - Get comments
  POST /api/journey/<id>/comments    - Add comment

EXTERNAL SERVICES:
  Claude API (claude-opus-4-20250805) for conversation
  PostgreSQL for data storage
  No other dependencies required

TECHNICAL SPECIFICATIONS
================================================================================

REQUIREMENTS:
  Python 3.8+
  Flask 2.0+
  SQLAlchemy
  anthropic>=0.25.0 (NEW)
  PostgreSQL (or MySQL with JSON support)

PERFORMANCE:
  Chat response: 2-5 seconds (Claude latency)
  Page load: <1 second
  Database queries: Indexed on user_id, slug, privacy
  Scalable to 10,000+ journeys

SECURITY:
  SQL injection: Prevented by SQLAlchemy parameterization
  XSS: Use Bleach/sanitizer for user input
  CSRF: Flask-WTF tokens on forms
  Private journeys: Validate share_token on access
  File upload: Validate type, size, rename (when added)

COST:
  Claude API: ~$0.02 per chat turn (Opus pricing)
  Database: Standard PostgreSQL costs
  Storage: < 1KB per journey in JSON format
  No other SaaS dependencies
