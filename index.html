What is Surgery Day Builder?

Surgery Day Builder is a complete, production-ready system that lets any user create beautiful, shareable healthcare journey timelines through conversational AI.

Instead of manually creating a Surgery Day page like Joe's, users can:

💬 Chat with AI

Describe your health journey conversationally. Claude asks clarifying questions about your condition, dates, and milestones.

🏥 Auto-Generate Timeline

AI automatically creates surgical stages, timeline entries, color schemes, and can generate medical diagrams.

✨ Tell Your Whole Story

Include clinical facts, faith moments, family support, and personal reflections — all organized beautifully.

🤝 Build Community

Others can comment, react with emojis (❤️ 🤗 💪 🎉), and share your journey with their networks.

Core Features

✅ Multi-Turn Chat

Natural conversation with Claude that maintains context across multiple turns.

📅 Timeline Pages

Beautiful, responsive timeline pages similar to the Surgery Day page.

🔒 Privacy Controls

Public, semi-private, or private journeys with share tokens and embed codes.

💾 Database Storage

Permanent storage in PostgreSQL with JSON fields for flexible data.

📸 Photo Gallery

Upload photos to S3 or local storage with AI-generated captions.

💬 Comments & Reactions

Community engagement with moderation and emoji reactions.

Quick Start (5 Minutes)

Test the demo immediately (no setup required):

pip install anthropic export ANTHROPIC_API_KEY="sk-ant-..." python3 demo_journey.py

Then have a conversation like:

"I had Chiari surgery on February 26, 2026. Recovery was 6 weeks."

Claude will ask follow-ups and generate a complete journey structure you can export to JSON.

System Architecture

How it works:

User Chat → Claude Multi-turn → Extract JSON → Save to DB → Publish Page ↓ Multi-turn Conversation (Gathers: condition, dates, milestones, faith/family elements) ↓ Extracts Structured Data (stages, timeline_entries, color_scheme) ↓ Creates Journey Page (/health-journey/<slug>)

Storage

During Chat: Flask Session or Redis (temporary, 24 hours)
After Publishing: PostgreSQL Database (permanent)
Photos/Videos: S3 or Local Filesystem (permanent)
Comments: PostgreSQL Database (permanent)

Database Tables

health_journeys - Journey metadata and AI-generated content
journey_comments - Visitor comments and reactions
