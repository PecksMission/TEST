WHAT GITHUB PAGES IS LOOKING FOR
================================================================================

GitHub Pages requires index.html at the ROOT of your repo.

CORRECT ✓
  your-repo/
  ├── index.html          ← HERE (root level)
  ├── README.md
  ├── models_*.py
  └── ...

INCORRECT ✗
  your-repo/
  ├── README.md
  └── docs/
      └── index.html      ← Wrong! Not at root

ALSO CORRECT ✓ (Alternative)
  your-repo/
  └── docs/
      └── index.html      ← Only if configured in Settings

For simplicity, put it at ROOT.

================================================================================
TROUBLESHOOTING
================================================================================

Still seeing 404 after upload?
  ✓ Check index.html is at ROOT (not in subfolder)
  ✓ Wait 3 minutes (not just 1-2)
  ✓ Clear browser cache (Ctrl+Shift+Delete)
  ✓ Try in a different browser

Homepage looks plain/unstyled?
  ✓ CSS is embedded in HTML, should work immediately
  ✓ Hard refresh browser (Ctrl+F5)
  ✓ Check Console for errors (F12)

Links don't work?
  ✓ Make sure all linked files are in your repo
  ✓ Links are case-sensitive on GitHub
  ✓ Try a different link to verify

Nothing changed after upload?
  ✓ GitHub might still be building (wait 5 min)
  ✓ Check Settings → Pages is enabled
  ✓ Confirm branch is correct (main/master)
