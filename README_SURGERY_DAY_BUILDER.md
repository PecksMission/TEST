# Surgery Day Builder - Complete Implementation

A system that lets any user create beautiful, shareable healthcare journey timelines through conversational AI.

## What It Is

Surgery Day Builder is an AI-powered feature that turns personal health stories into interactive timelines. Instead of manually creating a Surgery Day page (like Joe's), any user can:

1. Chat with Claude AI about their health journey
2. AI asks clarifying questions about the procedure, dates, and milestones
3. AI generates a structured timeline with stages, entries, and visual elements
4. System publishes a beautiful, shareable page like `/health-journey/chiari-surgery-recovery`
5. Others can comment, react with emojis, and share

## Architecture

### How It Works

```
User → Chat Interface → Claude API → Extract JSON → Save to DB → Publish Page
                      ↓
              Multi-turn Conversation
              (Gathers: condition, dates,
               milestones, faith/family elements)
                      ↓
              Extracts Structured Data
              (stages, timeline_entries, color_scheme)
                      ↓
              Creates Journey Page
              (/health-journey/<slug>)
```

### Data Flow

```
1. User Describes Journey
   "I had Chiari surgery Feb 26..."
   
2. Claude Asks Follow-ups
   "Any faith moments? Family support?"
   
3. User Responds
   "Yes, my faith was crucial..."
   
4. Claude Generates Structure
   ```json
   {
     "title": "Chiari Surgery Recovery",
     "condition": "Chiari Type I",
     "stages": [...],
     "timeline_entries": [...]
   }
   ```
   
5. System Publishes
   GET /health-journey/chiari-surgery-recovery
```

## Database Schema

```
health_journeys
├── id (UUID, PK)
├── user_id (FK to users)
├── slug (unique, indexed)
├── title, condition, procedure_type
├── start_date, end_date
├── status (draft/active/completed)
├── stages (JSON: [{name, description, color}])
├── timeline_entries (JSON: [{date, type, title, content, media}])
├── color_scheme (JSON: {clinical, faith, family, accent})
├── privacy (public/semi-private/private)
├── share_token
├── is_published, is_deleted
└── created_at, updated_at

journey_comments
├── id (UUID, PK)
├── journey_id (FK)
├── user_name, user_email
├── comment (text)
├── reactions (JSON: [{type, user, created_at}])
├── is_approved, is_deleted
└── created_at
```

## API Endpoints

### Public Routes
```
GET  /journey/builder                    # Chat interface

### API
```
POST /api/journey/chat                   # Process chat message
POST /api/journey                        # Create journey
PUT  /api/journey/<id>                   # Update journey

POST /api/journey/<id>/share             # Share with email
GET  /api/journey/<id>/comments          # Get comments
POST /api/journey/<id>/comments          # Add comment
POST /api/journey/<id>/comments/<id>/react # Add reaction
```

## Usage Examples

### Create a Journey

```bash
curl -X POST http://localhost:5000/api/journey/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "I had Chiari surgery on February 26, 2026",
    "session_id": "abc-123"
  }'
```

Response:
```json
{
  "response": "That sounds difficult. Tell me more about your recovery...",
  "journey_structure": null,
  "needs_more_info": true,
  "conversation_stage": "gathering",
  "turn": 1
}
```

After several turns with complete info:
```json
{
  "response": "Here's the journey structure I've created...",
  "journey_structure": {
    "title": "Chiari Surgery Recovery",
    "condition": "Chiari Type I",
    "stages": [...],
    "timeline_entries": [...]
  },
  "conversation_stage": "complete",
  "needs_more_info": false
}
```

### Publish Journey

```bash
curl -X POST http://localhost:5000/api/journey \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Chiari Surgery Recovery",
    "condition": "Chiari Type I",
    "start_date": "2026-02-26",
    "end_date": "2026-03-15",
    "stages": [...],
    "timeline_entries": [...],
    "color_scheme": {...},
    "privacy": "public"
  }'
```

Response:
```json
{
  "id": "uuid",
  "slug": "chiari-surgery-recovery",
  "public_url": "https://pecksmission.com/health-journey/chiari-surgery-recovery",
  "share_url": "https://pecksmission.com/health-journey/chiari-surgery-recovery?share_token=...",
  "embed_code": "<iframe src=\"...\"></iframe>"
}
```

## Troubleshooting

### Claude API Issues
```
Error: "ANTHROPIC_API_KEY not found"
→ Check .env file and load_dotenv() is called

Error: "Rate limit exceeded"
→ Wait a few seconds, implement queue in production
```

### Database Issues
```
Error: "No such table: health_journeys"
→ Run: flask db upgrade

Error: "Foreign key constraint failed"
→ Ensure user_id references valid user
```

### Template Issues
```
Error: "Template not found"
→ Verify templates/journey/ directory exists
→ Check template filenames match route references
```

## Performance

- **Chat Response Time**: ~2-5 seconds (Claude API latency)
- **Page Load Time**: <1 second
- **Comment Posting**: <500ms
- **Database Queries**: Indexed on user_id, slug, privacy, created_at

## Security

- **SQL Injection**: Prevented by SQLAlchemy
- **XSS**: Sanitize user input (use Bleach or similar)
- **CSRF**: Use Flask-WTF tokens
- **Private Journeys**: Validate share_token on every access
- **File Upload**: Validate file type, size, rename
